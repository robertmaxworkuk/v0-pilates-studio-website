# Фаза 1: Инфраструктура, Аутентификация и Ролевая Системы

В рамках первой фазы мы заложим архитектурную основу проекта, создадим базу данных и настроим базовую функциональность авторизации для проверки ролевого доступа.

## Цели Фазы
1. Подготовить и применить полный SQL-скрипт со схемой таблиц, триггерами и первоначальными RLS согласно мастер-плану. Настроить Supabase.
2. Создать страницы регистрации (`/sign-up`) и входа (`/sign-in`).
3. Настроить систему ролей. Администратора мы создаем вручную в Supabase Dashboard после регистрации.
4. Разработать базовые макеты (layouts) для клиентской (`/(user)/`) и административной (`/admin/`) панелей с маршрутизацией посредством Middleware для тестирования ролевого доступа.

## План работ
- [ ] 1. Инициализация и настройка Supabase клиентов (браузер / сервер).
- [ ] 2. Подготовка файла миграции (schema.sql) со всеми таблицами по мастер-плану (включая users_profile, sessions, bookings и т.д).
- [ ] 3. Написание триггера `on_auth_user_created` для автоматического копирования UUID из системной `auth.users` в публичную `public.users_profile` с базовой ролью `'user'`.
- [ ] 4. Создание клиентских страниц `/sign-up` и `/sign-in` (с валидацией через `zod` + `react-hook-form`).
- [ ] 5. Настройка Middleware `middleware.ts` для проверки сессии и ролевой маршрутизации.
- [ ] 6. Создание каркаса `app/admin/layout.tsx` и `app/(user)/layout.tsx` (и соответствующих вложенных страниц, например `/profile` и `/dashboard`) для проверки корректного входа и чтения профиля.
- [ ] 7. Проведение ручного тестирования логина, регистрации, подмены роли на `admin` в Supabase DB и проверки доступа к админской зоне.
